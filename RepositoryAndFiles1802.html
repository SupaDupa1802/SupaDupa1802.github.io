Creating, Navigating, and Deleting Repositories

Create a New Repository

At the main page after login
  1. Click Start a repository
  2. Enter name of the new repository in the textbox
  3. Choose public or private
  4. Click Create repository
  5. If necessary, click Settings to fine tune repository accessibility

Naviagate to a Repository

From the main page after log in
  1. The repositories are boxes with names in them
  2. Click on the name of repository
At some repository to go to another
  1. On top of the page is the current repository name
  2. Click on the name of the root repository
  3. At the root repository click name of new repository

Delete a Repository

At the repository to be deleted
  1. Click Setting
  2. At Setting, scrolled down to the bottom Danger Zone
  3. Click Delete this repository
  4. Follow instructions, confirm intention to delete, and provide the password
Note: A repository once deleted cannot be recovered

Assigning Repository Type
Accessibility can be defined when the repository is created
It can be changed anytime after creation
There are 3 level of accessibiltiy

The Private Repository

A private repository is visible and allows access only after login

To create a private repository

    Choose private when the repository is created

To Change an existing repository from public to private

  1. At the repository page, click Setting
  2. At Setting, scroll to Danger zone at bottom of page
  3. Click Visibility and change it to private
  4. Answer validating questions and confirm change

The Public Repository

A public repository is visible to all those coming to the account, even without logging in
It requires logging in to edit contents in public repositories

To create a public repository

    Choose public when the repository is created
To Change an existing repository from private to public
  1. At the repository page, click Setting
  2. At Setting, scroll to Danger zone at bottom of page
  3. Click Visibility and change it to public
  4. Answer validating questions and confirm change

The Server Repository

The server repository is similar tothe main io repository
  • Files in server repositry can be accessed anywhere on the www using the https protocol
  • After arranging with specific guests, the repository is able to receive files posted to it
To create a https accessible repository
  1. Use a public repository already created
  2. Load or create at least 1 file in the repository
  3. At the repository page, click on the Setting button
  4. At Settings, scroll to and click Pages in the panel on the left of page
  5. Under Branch, click on "None" box and change it to Main
  6. Click Save

Note: As this will be a very public repository great care should be exercise,
and no sensitive or vulnerable information should be place there

The accountname.io Repository (io)

The io is unique
  • There is only 1 io repository allowable in an account
  • It is used as the main server from the account to the www
  • From outside of the account, all other repositoryis accessed
    as if they are subfolders ofthe io repository
  • Files in io repositories can be accessed using the https protocol
Creating the io repository
  1. At the main Page, immediately after the account is created
  2. Create a public repository, but name it accountname.github.io
The io in SpupaDupa1802 is created as a public repository name SupaDupa1802.github.io

Moving Files

Uploading a file from Computer to SupaDupa1802

At the repository
  1. Click on the + button, then click on Upload a file
  2. Search file on the File Search panel that appears
  3. Mark the file then click Open
  4. When file loaded, click Commit Change

Downloading a file from Supadupa1802 to Computer

At the repository
  1. Click on the name of the file to download
  2. Click on the raw button (button with 3 dots near top right of page)
  3. Click Download File
Depending on browser setting, the file will be downloaded
either to the Download folder of your computer
or a download search panel will appear,
allow you to direct which folder to receive the file

Replicating (copy) or moving a File

  1. Download the file from the source repository to a temporary folder
    (usually the Download folder) on your computer
  2. If the file is to be copied to a different repository
    • Upload the file from the temporary folder to the destination repository
  3. If the file is to be replicated in the same reository
    • Rename the original file
    • Upload the file from the temporary folder backto the repository
  4. If the file is to be relocated to a different repository
    • Upload the file from the temporary folder backto the repository
    • Delete the original file

Edit, Create, and Delete Files

To Edit a File

At the repository holding the file
  1. Click on the file name to show file
  2. Click on the icon with the pencil (above and to the right of file listing)
  3. In the editing panel, change the file as required
  4. In the filename box, change the file name as required
  5. When editing finished, click the Commit Changes button

To Create a New File

A new file can be initiated by
  • Click on the + button, then click on Create a New File or
    Upload a template file from the computer
The file can then be edited by a combination of
  • direct typing in and changing the computer codes
  • Copying code snippets from available source and paste into the edit
The file can be renamed by editing the file name box

To Delete a File

At the repository
  1. Click on the file name
  2. Click on the account icon Top right corner of page
  3. Scroll to bottom of the drop down list
  4. Click Delete File
  5. Click Commit Changes (twice in different places on page)

To Clean a File's edit history

Github keeps a history of changes to a file since its creation.
This makes the file size large and contents confusing

When a file is finalized
Its a good idea to remove the editing history

To do so

  1. Download the file to the temporary folder in the computer
    Only the file code is downloaded, not the history
  2. Delete the file in the repository
    The history will also be deleted
  3. Upload the file from temporary folder to the repository
    The file is back, but history gone
